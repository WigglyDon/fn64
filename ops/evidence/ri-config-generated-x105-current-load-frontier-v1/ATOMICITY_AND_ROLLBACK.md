# Atomicity and rollback

Planning resolves instruction identity, old base, address, alignment, direct
target, old known source, low transfer word, defined-field policy, provenance,
and current control-flow context before mutation. The closed application plan
then performs one RI_CONFIG assignment followed by existing infallible cadence.

Unknown operands, non-direct or neighboring addresses, undefined high bits,
and target misses preserve GPR values/sources, HI/LO, COP0, all memory and SP
IMEM provenance, both RI facts, PC/next-PC, Count, and delay context. AdES owns
unaligned state and also leaves RI and memory unchanged. Tests compare complete
pre/post Machine snapshots for the bounded rejection paths.
