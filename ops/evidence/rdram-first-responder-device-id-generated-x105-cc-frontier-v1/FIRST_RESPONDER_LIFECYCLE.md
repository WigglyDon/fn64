# First-responder lifecycle

- Construction and complete cold-x105 bootstrap: unavailable.
- Exact accepted store: available with exact request/provenance.
- Second exact store: replaces the single fact and its provenance.
- Repeated complete bootstrap and general reset: cleared.
- Failed bootstrap: complete prior state preserved.
- Independent Machines: independent bytes, request fact, and provenance.

Immutable MI_VERSION remains unchanged through every lifecycle transition.
