# Atomicity and rollback

Complete-state snapshots prove atomic non-exception rejection for unsupported
words, unknown lineage, RCP 1.0 and neighboring apertures, pending MI transfer,
and repeated rejected use after prior success.

Unaligned ordinary and delay-slot stores take existing AdES code 5 with exact
BadVAddr, EPC, BD, exception vector, zero normal Count cadence, and no request
or sibling mutation. A successful ordinary delay-slot store uses existing
cadence exactly once.

The generated JAL rejection at `0xA40001A0` likewise preserves all CPU lineage,
delay context, memory, MI/RI/RDRAM facts, reservations, cartridge, and host
state.
