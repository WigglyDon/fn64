# Request versus module-state boundary

Represented: one guest CPU requested first-responder DEVICE_ID zero through the
exact RCP 2.0 aperture.

Unavailable: responder presence, responder identity/index, physical receipt,
assignment completion, current per-module DEVICE_ID, installed module count,
device type/manufacturer, capacity, readiness, and routing changes.

Request truth does not imply response truth.
