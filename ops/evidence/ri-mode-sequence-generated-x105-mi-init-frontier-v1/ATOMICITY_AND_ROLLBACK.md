# Atomicity and rollback

Planning resolves identity, old base, exact address, alignment, target, known
old source, low transfer word, defined-field policy, CPU provenance, and
control-flow context before mutation. The closed application plan then assigns
RI_MODE once and applies existing infallible cadence.

Unknown operands, non-direct or neighboring targets, and undefined high bits
preserve all GPR values/sources, HI/LO, COP0, RDRAM, SP memories/provenance,
all RI facts, PC/next-PC, Count, and delay context. Existing AdES owns
unaligned state and also leaves RI and memory unchanged. Failed bootstrap and
the MI_INIT_MODE frontier likewise expose no partial replacement.
