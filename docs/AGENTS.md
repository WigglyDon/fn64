# Documentation Scope Law

Scope: `docs/` and its descendants.
Inherits: [repository standing law](../AGENTS.md).
Owner: durable project context, decisions, protocols, and navigational truth.

Documentation may describe source and runtime evidence but may not promote a
claim beyond its evidence class. Mutable facts have one canonical owner and are
linked, not copied. Historical statements must be visibly historical.

Allowed dependencies flow from documentation to source anchors, Git evidence,
runtime artifacts, and canonical context owners. Product source, machine state,
and lane implementation authority must never depend on documentation prose.

Read [the context index](INDEX.md), [current state](context/CURRENT_STATE.md),
[the context-system law](context/README.md), and the relevant subsystem page.
Current delivery-process truth belongs in
[MASTER_DIRECT_WORKFLOW.md](process/MASTER_DIRECT_WORKFLOW.md). Retired
Worker-topology history remains in
[WORKTREE_PROVISIONING.md](process/WORKTREE_PROVISIONING.md), not lane prose.
Validate with `tools/fleet/context-verify` once available. Stop on conflicting
current owners, unresolved authority, or evidence that disproves current text.

Update this file only when the documentation authority boundary changes.
