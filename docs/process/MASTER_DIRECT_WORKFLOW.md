# Master Direct Workflow

Context role: canonical current delivery workflow.
Scope: product work from decision through repository integration and reporting.
Canonical for: current roles, direct-pass Git authority, transport, and retirement of the former fleet operation.
Not canonical for: product behavior, current capability, or historical Worker results.
Inherits: [repository standing law](../../AGENTS.md).
Current-state owner: [CURRENT_STATE.md](../context/CURRENT_STATE.md).
Related evidence: [lane registry](../lanes/lane-registry.md), [packet protocol](PACKET_PROTOCOL.md), and [worktree provisioning](WORKTREE_PROVISIONING.md).
Update triggers: a change to the active delivery roles or direct-pass authority.

## Current loop

The current project loop is:

`Don -> Master GPT -> Master Codex -> Master GPT`

Master GPT owns architecture, sequencing, scope, and interpretation of results.
Master Codex directly owns live repository inspection, bounded implementation,
validation, commits, Master-worktree mechanics, canonical integration,
authorized normal push, evidence packaging, and the complete final report. Don
transports one complete Master packet in and one complete Master report out; he
does not perform routine Git, conflict, validation, or prompt-construction work.

## Retired fleet operation

Supervisor GPT and Worker Codex lane roles are retired. No new supervisor seed,
Worker packet, Worker lane, Worker branch, Worker worktree, or Worker process is
created or launched. The integration queue therefore contains no active Worker
candidate.

Existing supervisor/Worker process pages, packet schemas, branches, worktrees,
commits, lane pages, and artifacts are historical infrastructure. They remain
available for audit and diagnostics, but carry no current execution authority.
Fleet tools remain dormant, non-product diagnostics and may still validate
context, queue, packets, evidence, and topology when useful.

Retirement does not accept unfinished work. A preserved branch or worktree is
historical donor state only unless a later explicit decision re-establishes
authority. Donor state is inspected read-only, is not merged merely because it
exists, and is never reset, cleaned, deleted, or repurposed automatically.

## Direct-pass safety

Each direct Master pass binds one literal Master worktree, branch, accepted
base, Context-SHA, scope, private-input authority, stop conditions, validation
contract, integration authority, and reporting envelope. Master Codex verifies
those facts before mutation, preserves unknown work, and uses non-destructive
Git mechanics. Normal push requires explicit packet authority, remote freshness,
no force, and complete validation.

Unsupported product behavior remains explicit. A direct workflow changes who
does the work; it does not broaden product, legal, private-input, deployment, or
compatibility authority.
